\n# 한국의 암벽등반지 — 웹사이트 프로젝트 기획서 (개발용)

> 이 문서는 Claude Code로 실제 홈페이지 구현을 시작할 때 참고할 기획/요구사항 정리본입니다.

## 1. 프로젝트 개요

- **제목**: 한국의 암벽등반지
- **한 줄 소개**: 한국의 암벽등반지 정보를 한곳에서!
- **핵심 컨셉**: 지도 기반 드릴다운(drill-down) 탐색 + 루트별 개념도/난이도/최신 후기/사진을 검색엔진처럼 한눈에 제공

## 2. 핵심 탐색 구조 (지도 드릴다운, 5단계)

한국 지도에서 시작해 클릭할 때마다 지도가 확대되며, 아래로 갈수록 정보가 구체화된다.

| 단계 | 화면 | 표시 내용 | 예시 |
|---|---|---|---|
| 0 | 전국 지도 | 등반지가 있는 지역 마커 | 서울, 경기도, 강원도 등 |
| 1 | 지역 지도 | 해당 지역의 등반지(산) | 북한산, 도봉산, 관악산 등 |
| 2 | 등반지(산) 지도 | 세분화된 루트 구역(봉우리) | 인수봉, 노적봉, 만경대 등 |
| 3 | 봉우리 | 루트 목록 | 인수A, 인수B, 취나드A, 취나드B, 우정A, 우정B, 비둘기길, 고독길 등 |
| 4 | 루트 상세 | 개념도, 난이도, 최신 후기/사진, 외부 링크 | 취나드B (1피치~5피치) |

**계층 경로 예시**: 서울 → 북한산 → 인수봉 → 취나드B(1~5피치)

각 단계는 지도 확대(zoom-in) + 이전 단계로 돌아가는 내비게이션(breadcrumb)을 지원해야 함.

## 3. 화면별 상세 기획

### 3-1. 메인 화면 (0단계 — 전국 지도)
- 한국 전도(全圖) 배경
- 등반지가 있는 지역에 마커/하이라이트 표시
- 지역 클릭 → 해당 지역으로 확대 이동

### 3-2. 지역 지도 (1단계)
- 클릭한 지역 확대 표시
- 지역 내 등반지(산) 마커 표시
- 등반지 클릭 → 다음 단계로 확대

### 3-3. 등반지 지도 (2단계)
- 등반지(산) 지도 확대 표시
- 세분화된 봉우리 표시
- 봉우리 클릭 → 루트 목록으로 이동

### 3-4. 루트 목록 (3단계)
- 선택한 봉우리의 전체 루트 리스트
- 각 루트에 간단 요약(난이도 등) 노출 권장

### 3-5. 루트 상세 (4단계, 최종)
표시 우선순위:
1. **개념도(topo)** — 가장 먼저 표시
2. **최신 사진**
3. **관련 최신 게시물 링크** (검색엔진처럼 큐레이션): 네이버 블로그, 유튜브 영상, 인스타그램 게시글
4. 기타 정보: 난이도, 피치 수(예: 1피치~5피치)

## 4. 데이터 모델

계층: **지역 → 산(등반지) → 봉우리 → 루트**

| 레벨 | 예시 | 필드(안) |
|---|---|---|
| 지역(Region) | 서울, 경기도, 강원도 | id, name, mapCoordinates/boundary, mountainIds[] |
| 산/등반지(Mountain) | 북한산, 도봉산, 관악산 | id, name, regionId, mapCoordinates, peakIds[] |
| 봉우리(Peak) | 인수봉, 노적봉, 만경대 | id, name, mountainId, coordinates, routeIds[] |
| 루트(Route) | 인수A, 취나드B | id, name, peakId, pitchCount, grade(난이도), topoImageUrl, latestPhotos[], externalLinks{naver, youtube, instagram} |

예시 JSON 스키마:

```json
{
  "region": { "id": "seoul", "name": "서울" },
  "mountain": { "id": "bukhansan", "name": "북한산", "regionId": "seoul" },
  "peak": { "id": "insubong", "name": "인수봉", "mountainId": "bukhansan" },
  "route": {
    "id": "chouinard-b",
    "name": "취나드B",
    "peakId": "insubong",
    "pitchCount": 5,

    }
  }
}
```

## 5. 핵심 기능

- 지도 기반 드릴다운 내비게이션 (전국 → 지역 → 산 → 봉우리 → 루트), 확대/이동(zoom & pan)
- 루트별 개념도 뷰어
- 난이도 표시 체계 (등급 체계 확정 필요 — 6절 참고)
- 최신 후기/사진/영상 링크 큐레이션 (수동 등록 또는 검색 API 연동)
- 검색 기능 (지역/산/봉우리/루트명으로 바로 검색) — 우선순위 검토 필요

## 6. 데이터 수집처 (조사 결과)

콘텐츠(개념도, 사진, 최신 글 링크) 확보를 위해 활용 가능한 출처:

1. **한국산악아카이브** (https://mtarchive.kr) — 2026.07.07 공식 오픈. 국내 최대 규모 산악 기록 디지털 아카이브(1900년대 초~현재, 약 1만 2,900여 종). 등반계획서·등반보고서·가이드북·사진 등에 개별 루트 단위 개척 기록도 존재 (예: "1986 한국은정산악회 북한산 인수봉 은정길 개척 등반 보고서"). 루트 개척사·개념도 원자료 확보에 가장 유용.
2. **대한산악연맹**(https://kaf.or.kr) 및 산하/유관 단체 — 한국산악회, 한국산서회, 한국대학산악연맹, 국립산악박물관(komount.or.kr), 지역 학생산악연맹 등.
3. **국립공원공단**(knps.or.kr) — 북한산·설악산 등 국립공원 내 등반 통제구역, 등반 신고 제도 등 공식 정보. 지도/구역 데이터의 기준점으로 활용.
4. **네이버 블로그 / 유튜브 / 인스타그램** — 최신 후기·사진·영상. 자동 수집 시 각 플랫폼 공식 검색/데이터 API 약관 확인 필요 (특히 인스타그램은 API 제약이 큼).
5. **국제 크라우드소스 등반 DB** (thecrag.com, 8a.nu 등) — 일부 한국 루트 정보 존재 (최신 상태는 별도 확인 필요).
6. **국내 등반 가이드북(서적)** — 예: 인수봉/설악산 등반 가이드. 저작권 있는 자료이므로 그대로 전재 불가 — 참고하여 직접 개념도 재작성 필요.

## 7. 결정 필요 사항 (Open Decisions)
